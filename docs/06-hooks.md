# Hooks reference

Every public hook the SDK exposes. All hooks must be called inside `<MiriAppProvider>`.

Hooks fall into three loose categories:

- **Context hooks** — direct access to a React context (`useCareSeeker`, `useProgram`, `useTheme`, …).
- **Query hooks** — TanStack Query-backed data fetchers with cache + refetch (`useWellnessScore`, `useInsights`, …).
- **API / repository hooks** — imperative wrappers that return functions you call manually (`usePriorityActionAPI().completePriorityAction()`).

## Identity

### `useCareSeeker()`

The current end-user's identity.

```ts
const { careSeeker, refresh } = useCareSeeker();
// careSeeker?.id           — Miri care_seeker_id
// careSeeker?.display_name — "Maya R."
// careSeeker?.first_name   — "Maya"
```

### `usePractitioner()`

The active practitioner (coach / clinician). Mostly relevant for branding the chat surface.

```ts
const { practitioner } = usePractitioner();
// practitioner?.id
// practitioner?.display_name
// practitioner?.chat_image    — used by <PractitionerHeader>
// practitioner?.productSku   — falls back to this when computing scope
```

### `useProgram()`

The user's active program (which SKU they're enrolled in). For multi-program platforms.

```ts
const { program, reload } = useProgram();
// program?.id
// program?.productSku  — 'multi_program', 'gut_health', 'nutritionist', etc.
// reload() — force a refresh after the user switches programs
```

## Scores & insights

### `useWellnessScore()`

```ts
const { data, isLoading, refetch } = useWellnessScore();
// data?.score             — composite Wellness Score
// data?.lever_scores      — array of per-lever scores
// data?.composite_scores  — HabitScore / ResponseIndex / WellnessScore
```

### `useWellnessScoreHistory({ days })`

Time-series of composite scores.

```ts
const { data } = useWellnessScoreHistory({ days: 14 });
```

### `useInsights()`

Narrative insights generated by the Layer 3 narration engine. Keyed by scope.

```ts
const { data, refetch } = useInsights();
// data?.daily_plan        — { text, generated_at, tight_text, citations }
// data?.weekly_review
// data?.pattern_analysis
// data?.recomputation_status  — 'idle' | 'in_progress'
// data?.awaiting_checkin
```

### `useLeverConfigs()`

The lever schema for the active SKU — what each lever measures, its target, its unit. Mostly used internally by `<LeverBreakdown>` / `<KeySignalsRow>`.

```ts
const { data } = useLeverConfigs();
// data?.[i] — { lever_id, label, unit, target_value, category, … }
```

## Priority Actions

### `usePriorityActionAPI()`

```ts
const { getPriorityAction, completePriorityAction } = usePriorityActionAPI();

const pa = await getPriorityAction();
// pa.action_text
// pa.rationale
// pa.is_completed
// pa.generated_at

const result = await completePriorityAction();
// result.reinforcement_text  — what to show after the user marks complete
```

## Coach / chat

### `useCoachChipsAPI()`

```ts
const { getCoachChips } = useCoachChipsAPI();
const { chips, subtitle } = await getCoachChips(sku);
// chips[i] = { chip_id, label, module_name, prompt_text, domain }
```

### `useChatSessionsAPI()`

```ts
const { sessions, createSession, fetchHistory } = useChatSessionsAPI();
// sessions: ChatSession[]   — prior conversations
// createSession({ moduleName, sendUserMessage? })
// fetchHistory({ chatSessionId })
```

### `useCoachConversationsAPI()`

Lower-level: list / search past conversations beyond the most recent sessions.

```ts
const { listConversations, search } = useCoachConversationsAPI();
```

## Habits

const { getHabitProgress, getStreaks } = useHabitProgressAPI();
const progress = await getHabitProgress({ habitId });

## Meals & nutrition

### `useMealsRepository()`

CRUD on meal entries.

```ts
const { listMeals, getMeal, updateMeal, deleteMeal } = useMealsRepository();
const meals = await listMeals({ date });
```

### `useNutritionRepository()` / `useNutritionAPI()`

Daily nutrition rollups + macro / micro nutrient breakdowns.

```ts
const { getDailyNutrition } = useNutritionRepository();
const totals = await getDailyNutrition({ date });
// totals.protein_g, totals.carbs_g, totals.fat_g, totals.fiber_g, …
```

### `useBarcodeAPI()`

```ts
const { lookupBarcode } = useBarcodeAPI();
const food = await lookupBarcode('012345678905');
```

### `useFoodImageAPI()`

```ts
const { analyzeFoodImage } = useFoodImageAPI();
const result = await analyzeFoodImage({ imageUri });
```

### `useVoiceMealParse()`

```ts
const { parseVoiceMeal } = useVoiceMealParse();
const meal = await parseVoiceMeal({ transcript: '…' });
```

## Quick check-in

### `useQuickCheckinRepository()`

Persistence layer for `<QuickCheckinFlow>` / `<DailyCheckIn>` cards.

```ts
const { submitQuickCheckinCard, getQuickCheckinCards } = useQuickCheckinRepository();
await submitQuickCheckinCard({ artifactType: 'medication_tracking_item', data });
const cards = await getQuickCheckinCards();
```

## Weight & body

### `useWeightHistory({ days })`

Day-by-day weight history (gaps included as nulls).

```ts
const { data, refetch } = useWeightHistory({ days: 7 });
// data?.history[i] = { date, value: number | null }
// data?.unit
```

### `useWeightGoalsRepository()`

CRUD on weight goal artifacts.

```ts
const { getWeightGoal, updateWeightTracking } = useWeightGoalsRepository();
```

### `useActiveMedicationGoal()`

The user's currently active medication, if any.

```ts
const { data } = useActiveMedicationGoal();
// data?.medication_name, data?.dose, data?.next_dose_at, …
```

## Tracking items (generic)

### `useTrackingItemGoalsRepository()`

For any tracked goal (water, sleep, activity, etc.).

```ts
const { getTrackingItemGoals, upsertTrackingItemGoal } = useTrackingItemGoalsRepository();
```

### `usePlanArtifactsAPI()`

Lower-level: any plan artifact by type / id.

```ts
const { listPlanArtifacts, getPlanArtifact, postPlanArtifact } = usePlanArtifactsAPI();
const artifacts = await listPlanArtifacts({ type: 'WEIGHT_TRACKING_ITEM', limit: 30 });
```

## App-level helpers

### `useMiriApp()`

Catch-all for cross-cutting state on the app shell.

```ts
const {
  getWeightProgress,       // composite weight + body-comp helper
  getBodyComposition,
  setPendingCameraImages,  // for the meal-photo capture flow
} = useMiriApp();
```

### `useBaseProvider()`

Direct access to the props you passed to `<MiriAppProvider>` (`apiKey`, `logError`, `showMessage`, `trackEvent`, etc.).

### `useTheme()`

Merged theme colors + fonts (see [Theming & branding](./04-theming.md)).

### `useBottomSheet()`

```ts
const { open, close } = useBottomSheet();
open({ component: <YourSheet />, snapPoints: [350, '80%'] });
```

## Naming conventions

- `…API()` — imperative hooks that return functions (e.g. `getCoachChips()`).
- `…Repository()` — imperative hooks that wrap a CRUD-style data layer.
- `use<NounContext>()` — context-reading hooks (`useCareSeeker`, `useProgram`).
- `use<Noun>()` — TanStack Query-backed hooks that return `{ data, isLoading, refetch, … }`.

If you find yourself wanting a hook that isn't here, autocomplete from `@miri-ai/miri-react-native` will surface anything we've exported — or [file an issue](https://github.com/mirihealth/miri-react-native-sdk-examples/issues) and we'll point you at it (or add it).
